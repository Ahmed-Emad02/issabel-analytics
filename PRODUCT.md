# Product

## Register

product

## Users

Issabel and Asterisk administrators, call-center supervisors, and operations staff use SPT-ANALYTICS while monitoring live PBX activity, reviewing call records, managing extensions, and checking GSM dongle state. They need fast scanning, reliable controls, and clear status feedback during support and operations work.

## Product Purpose

SPT-ANALYTICS provides a real-time operational dashboard for Issabel 5 and Asterisk 18. It brings CDR analytics, extension statistics, live operator monitoring, GSM dongle status, address book management, user permissions, and recording upload into one authenticated interface. Success means operators can find call activity, spot line state, and complete routine PBX tasks without guessing where to click.

## Brand Personality

Precise, operational, controlled. The interface should feel like a dependable telecom control surface: dark by default, high contrast, restrained, and direct.

## Anti-references

Do not turn the dashboard into a marketing site or decorative analytics demo. Avoid changing the existing red, black, white, and zinc theme colors. Avoid colorful novelty palettes, unnecessary animation, oversized rounded cards, soft-glow effects, and copy that sounds like generic SaaS promotion. Do not change backend behavior while polishing UI.

## Design Principles

1. Preserve operational trust by keeping familiar dashboard patterns and consistent controls.
2. Prioritize scanability for tables, live status cards, and dense call metrics.
3. Use the existing theme as identity; improve hierarchy through spacing, contrast, and state clarity.
4. Make responsive layouts structural so operators can use the dashboard on small maintenance screens.
5. Treat motion as state feedback only, with reduced-motion support.

## Accessibility & Inclusion

Target WCAG AA contrast for body text, controls, placeholders, focus states, and table metadata. Support keyboard focus visibility across navigation, filters, tables, audio controls, and modals. Preserve English and Arabic/RTL support. Respect reduced-motion preferences for hover, pulse, modal, and chart-style transitions where CSS controls apply.
